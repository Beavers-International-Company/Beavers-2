# Project Info

## Course

IT Project Management

## Project Name

Smart Scooter Campus

## Project Type

Educational MVP web project.

## Project Concept

Smart Scooter Campus is a simple campus e-scooter rental web service prototype. The project demonstrates how a student can register with a student email, view available scooters, see battery levels, start a rental through a simulated QR scan, finish the ride, and receive eco bonuses.

## Project Management Context

- **Scrum:** The project is organized around sprint-based delivery and a prioritized backlog.
- **Jira:** Sprint 1 tasks use Jira-style issue keys such as SCRUM-10, SCRUM-15, SCRUM-16, and SCRUM-20.
- **Confluence:** This document can be used as a lightweight project knowledge base page.
- **MVP:** The first version implements only the most important user flow with mock data.
- **Student project:** The project is intentionally simple and suitable for learning web development and IT project management.

## Sprint 1 MVP Scope

| Jira Key | Task | Story Points | MVP Result |
| --- | --- | ---: | --- |
| SCRUM-10 | Student email registration | 5 SP | Student-domain validation and verified profile |
| SCRUM-15 | View available scooters on the map | 8 SP | Mock campus map/list with scooter cards and availability filter |
| SCRUM-16 | Display scooter battery level | 3 SP | Battery percentage displayed for each scooter |
| SCRUM-20 | QR-code scanning simulation to start rental | 8 SP | Scooter selection starts a simulated ride |

## Limitations

- No real backend.
- No database.
- No real authentication.
- No real GPS or Google Maps integration.
- No real QR scanner.
- No payment system.
- All data is mock data or stored in browser localStorage.
